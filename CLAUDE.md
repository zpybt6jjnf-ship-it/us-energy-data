# US Energy Data Platform

Public-facing data platform making US energy data accessible. Based on Hannah Ritchie's proposal (Feb 2026). Inspired by Our World in Data. 

## Tech Stack
- **Frontend**: SvelteKit 5 + TypeScript (strict) + Tailwind CSS v4 + Cloudflare adapter
- **Charts**: D3 (scales, shapes, geo, interpolate) + Svelte reactive SVG — D3 is a utility library only, Svelte handles all DOM
- **Maps**: d3-geo (AlbersUSA) + topojson-client + us-atlas (`states-10m.json`, NOT `states-albers-10m.json`)
- **Typography**: Google Fonts — Instrument Serif (display), DM Sans (body), JetBrains Mono (data)
- **Data Pipeline**: Python (requests + pandas + openpyxl) → static JSON → `site/static/data/`
- **Hosting**: Cloudflare Pages

## Architecture Decisions
- **Pipeline and site are decoupled**: Python pipeline → JSON → SvelteKit prebuild. Site never calls EIA API at runtime. Pipeline stages are independent — partial failures export what succeeded. Safe to re-run.
- **URL is single source of truth** for chart state: `/prices?state=TX,CA&sector=residential` (managed by `chartConfig.ts` store)
- **Static JSON per chart** in `site/static/data/` (36 files across 7 categories). No database, no API server. JSON schemas must match TypeScript types in `site/src/lib/types/data.ts` — update both when changing pipeline output.
- **Brand-first color system**: Custom `@theme` tokens (`bg-primary`, `text-accent`, `bg-surface-alt`, etc.) replace default Tailwind palette. Never use default gray/blue.
- **Editorial page design**: Serif headlines → explanatory intro → chart sections with alternating backgrounds → source attribution + download buttons

## Chart Conventions
- **LineChart**: End-of-line labels, hover crosshair + dots, no rotated Y-axis labels (units in subtitle)
- **BarChart**: Hover dimming, multi-color legend, vertical or horizontal
- **Scatter**: Hover point enlargement + label, white stroke separation
- **ChoroplethMap**: State abbreviation labels (`FIPS_TO_ABBR` in `states.ts`), cream-to-navy gradient via d3-interpolate
- **ChartWrapper**: `hero` prop for featured charts, collapsible "About this chart" descriptions, ghost-style CSV/PNG download buttons
- **Fuels special case**: "All Fuels" view uses base year = 100 index; single fuel uses raw values with proper units
- **Spacing**: `mt-10` after header, `mt-8` between charts, `mt-16` for major sections

## Data Sources & EIA API Endpoints
- **EIA API v2** (`https://api.eia.gov/v2/`): prices, demand, generation, capacity, emissions, fuels, trade
- **EIA-861**: Reliability (SAIDI/SAIFI/CAIDI) — ZIP downloads, not API. URL pattern: `eia.gov/.../archive/zip/f861{year}.zip`
- **Census Bureau**: State population estimates (static 2023 data in `census.py`)
- **BLS CPI-U**: Inflation adjustment (static table in `bills.py`)

Key API quirks:
- Natural gas: use `/natural-gas/prod/whv/data` (not `/sum/lnd/`)
- Crude oil: use `/petroleum/crd/crpdn/data`
- EIA-861 2019: has extra "Short Form" column that must be handled
- EIA-861 "." values mean NaN
